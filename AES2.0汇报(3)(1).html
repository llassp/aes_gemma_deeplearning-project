核心直觉

将 k 分类转化为 k−1 个独立二分类，共享 backbone，决策边界独立。
每个切点问："这篇作文是否 > k 分？"→ 模型无法用"猜均值"逃避

核心发现

序数回归 = 防坍缩的必要条件。没有序数 → 无论 Pooling 多好 → 必崩
单折 Eval 完全不可信：Exp 2 单折 0.846 却 OOF 0.024

Huber 回归的数学陷阱

∥ 最优预测器满足： f*(x) = E[Y | X=x] （条件期望）

∥ 数据不均衡下 → E[Y|X] ≈ 全局均值 3.5

∥ 模型“理性地”输出 ~3.5 → 预测分布完全坍缩

∥ 结论：回归损失在极端不均衡数据上本质性失效

改进 2：Attention Pooling — 从 Last‑Token 到可学习加权

Last‑Token Pooling 的局限

仅取序列最后一个非 padding token，丢弃了前面 1500+ tokens 中散布的信息

t₁t₂t₃...t₁₅₀₀t_last ✓

↑ 只取最后 1 个 token，其余全部丢弃

Attention Pooling 设计

Linear(H→H) → LayerNorm → GELU
→ Linear(H→1, zero-init)
→ Softmax → 加权求和
w₁=0.12w₂=0.08w₃=0.05w₄=0.18wₙ=0.25

失败尝试与反思 — 我们踩过的坑

🚫

失败 0：CoT + SFT + DPO — 整条路线的放弃

CoT 思维链生成：要求 4B 模型同时推理打分过程 → 注意力分散于格式学习 → QWK 等同随机。SFT 微调：逐 token 生成标签文本 → 单折 QWK ~0.6，预测集中于 3-4 分。DPO：SFT 失败后自然失去价值。教训：回归本质——目标是输出分数，不是生成文本。

💥

失败 1：Huber + Attention Pooling — 0.846 是你的谎言

单折 0.846（最高），OOF 0.024（最低）。更强记忆 + 错误 Loss = 更精确地输出均值 3.5。教训：单折 eval 完全不可信，OOF 是唯一金标准。

DDP 设备同步 + 两阶段 OOM

pos_weight CPU/GPU mismatch → pos_weight.to(logits.device)；Stage 2 参数量增大 → A100 OOM → batch_size 4→2

工程踩坑集

Gemma 4 ClippableLinear → PEFT 保存失败 → Monkey Patch | FlashAttention-2 + BF16 兼容性 | Windows → num_workers=0
