> **赛题**: Kaggle AES 2.0 — Automated Essay Scoring
> **队名**: 【待填写】
> **基线**: V6 (Gemma4 + LoRA + Huber Regression + Last-Token Pooling)
> **完整方案**: V8 (Ordinal Regression + Attention Pooling + 5-Fold OOF)

## 一、消融实验设计（≥4 组 → 加分项）

### 架构总览

```
V6 Baseline (Huber + Last-Token)
    │
    ├── V8-Full: Ordinal + Attention Pooling + pos_weight + Two-Stage + Data Balance + OOF Threshold
    │
    ├── Ab1: w/o Ordinal (V6 Huber head, 1-dim, keep Attn Pooling)
    ├── Ab2: w/o Attention Pooling (V8 Ordinal head, but last-token pooling)
    ├── Ab3: w/o pos_weight (V8 Ordinal, but plain BCE)
    ├── Ab4: w/o Two-Stage Training (single-stage joint training from start)
    └── Ab5: w/o Data Balancing (no sample weights, no oversampling)
```

### 1.1 实验矩阵

| # | 实验名称 | Pooling | Loss Head | pos_weight | Two-Stage | Data Balance | OOF阈值 |
|---|---------|---------|-----------|------------|-----------|-------------|---------|
| 0 | **Baseline (V6)** | Last-token | Huber (dim=1) | — | — | — | — |
| 1 | **V8-Full** | Attn | Ordinal (dim=5) | ✅ | ✅ | ✅ | ✅ |
| 2 | w/o Ordinal | Attn | Huber (dim=1) | — | ✅ | ✅ | ✅ |
| 3 | w/o Attn Pooling | Last-token | Ordinal (dim=5) | ✅ | ✅ | ✅ | ✅ |
| 4 | w/o pos_weight | Attn | Ordinal (dim=5) | ❌ | ✅ | ✅ | ✅ |
| 5 | w/o Two-Stage | Attn | Ordinal (dim=5) | ✅ | ❌ | ✅ | ✅ |
| 6 | w/o Data Balance | Attn | Ordinal (dim=5) | ✅ | ✅ | ❌ | ✅ |

### 1.2 各组实验的具体修改

#### Exp 0: Baseline (V6)
- 直接使用服务器上已有的 V6 代码和模型
- `sft_regression_v6.py` + Huber Loss + last-token pooling
- 无 pos_weight、无 Two-Stage、无 OOF 阈值优化
- **复用已有结果**: OOF QWK ≈ 0.8101

#### Exp 1: V8-Full（完整方案）
- 已跑完：`sft_ordinal_v8.py`
- OOF QWK = **0.8354**
- 阈值: `[1.63, 2.57, 3.59, 4.77, 5.40]`

#### Exp 2: w/o Ordinal（验证序数回归的贡献）
- 以 V8-Full 为基础，将 `num_labels` 从 5 改回 1
- Loss 从 `BCEWithLogitsLoss` 改回 `HuberLoss`
- 保留 Attention Pooling、Two-Stage、Data Balance
- **修改点**: `Gemma4ForScoreRegression.__init__` 中 `num_labels=1`
- **修改点**: `forward()` 中 loss 计算回 Huber
- **修改点**: `compute_metrics()` 回 V6 版本

#### Exp 3: w/o Attention Pooling（验证注意力池化的贡献）
- 以 V8-Full 为基础，移除 `AttentionPooling` 模块
- 回退到 last-token pooling
- **修改点**: 删除 `self.pooler = AttentionPooling(hidden_size)`
- **修改点**: `forward()` 中改为 last-token 取法
- **修改点**: `modules_to_save` 移除 `"pooler"`

#### Exp 4: w/o pos_weight（验证类别平衡损失权重的贡献）
- 以 V8-Full 为基础，`pos_weight` 设为 `None`
- BCE loss 不对正样本（高分边界）加权
- **修改点**: `forward()` 中 `_pw = None`（固定值）

#### Exp 5: w/o Two-Stage（验证渐进训练策略的贡献）
- 以 V8-Full 为基础，跳过 Stage 1，直接 joint training
- **修改点**: 配置中 `stage1_epochs: 0`
- LoRA 和 head/pooler 从一开始就同时训练

#### Exp 6: w/o Data Balance（验证数据平衡策略的贡献）
- 以 V8-Full 为基础，不做 per-sample weighting 和 oversampling
- **修改点**: 配置中 `use_weighted_loss: false`, `use_oversampling: false`

### 1.3 跑实验计划

考虑到 5-fold 完整训练耗时较长（~1.5h/fold），建议使用 **单折 (Fold 0)** 进行消融实验，用验证集 QWK 作为代理指标。最终 OOF QWK 只在 Exp 0 和 Exp 1 报告。

| 实验 | 估计时间 | 优先级 |
|------|---------|--------|
| Exp 0 (Baseline) | 已有结果 | ✅ |
| Exp 1 (V8-Full) | 已完成 | ✅ |
| Exp 2 (w/o Ordinal) | ~1.5h | 🔴 高 |
| Exp 3 (w/o Attn) | ~1.5h | 🔴 高 |
| Exp 4 (w/o pos_weight) | ~1.5h | 🟡 中 |
| Exp 5 (w/o Two-Stage) | ~1.2h | 🟡 中 |
| Exp 6 (w/o Data Balance) | ~1.5h | 🟢 低 |

**推荐优先跑 Exp 2 + Exp 3**（分别验证 Ordinal 和 Attention Pooling 的核心贡献），加上已有 Exp 0/1 共 4 组，刚好满足 4 组消融实验的加分要求。
