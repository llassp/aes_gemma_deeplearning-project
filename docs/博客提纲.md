## 二、文章结构

### Part 1：问题引入（~500字）

- Kaggle AES 2.0 赛题简介：1-6 分作文自动评分
- 评估指标 QWK 的含义
- 一句话预告核心发现：你信的单折 eval 可能是假的

### Part 2：Baseline 搭建（~600字）

- 基座选择：Gemma 4 E4B + LoRA（为什么）
- V6 Baseline: Huber 回归 + Last-Token Pooling
- 单折 eval QWK = 0.8101，看起来很稳
- 但这里的"验证集"其实跟训练集同源……

### Part 3：V8 升级思路（~800字）⭐ 重点

- 三个改进方向：
  1. **序数回归**：1-6 分 → 5 个独立二分类切点（配图）
  2. **Attention Pooling**：替换 last-token，学习全局加权
  3. **5-Fold OOF + 动态阈值**：无偏估计
- 架构对比图（V6 vs V8）

### Part 4：消融实验——三个实验，三个核心发现（~1000字）⭐ 最重点

| 实验 | Pooling | Loss | 单折 QWK | OOF QWK |
|------|---------|------|---------|------------|
| V8-Full | Attn | Ordinal BCE | 0.836 | **0.835** |
| Exp 2 | Attn | Huber | **0.846** | **0.024** ←炸了 |
| Exp 3 | Last-Tok | Ordinal BCE | 0.831 | 🔄 |

**发现 1：单折 eval 完全骗人**

Exp 2 单折 0.846（最好），OOF 0.024（最差）。Huber + Attn Pooling 在训练集上学得太好，面对没见过的作文直接输出全局均值 3.5。

```
Exp 2 OOF 预测分布：
  所有 17,307 篇作文的预测值
  min=3.34, max=3.68, std=0.05
  模型：不认识 → 猜 3.5 分
```

**发现 2：序数回归是唯一解**

V8-Full OOF std=1.06，完整覆盖 1-6 分区间。5 个独立二分类迫使模型对每个切点做出判断，无法"猜均值"。

**发现 3：Attention Pooling 锦上添花**

Ordinal 解决坍缩问题后，Attn Pooling 再 +0.004（0.831→0.835）。

### Part 5：失败记录——那些没 work 的尝试（~400字）

- PEFT + Gemma4 保存 bug（Monkey Patch 修复过程）
- FlashAttention-2 + DDP 的坑（4 个崩溃 bug 逐个解）
- `pos_weight` 设备不匹配（CPU vs GPU）
- 两阶段训练中 Stage 1→2 的 OOM（从 batch=4 降到 2）
- Screen 会话被 SSH 断连 kill（systemd-run 保护）

### Part 6：智能体（Claude Code）使用记录（~300字）

- 全部代码由 Claude Code 在人工监督下生成和调试
- 一个 Prompt 完成 V6→V8 架构设计（Prompt 原文附上）
- 迭代了约 10 轮修复运行时 bug
- 消融实验方案由 Claude Code 读取作业要求后自动设计编排

### Part 7：总结与 Takeaway（~300字）

- 单折 eval ≠ 模型泛化能力（最重要的一课）
- Huber/MSE 回归在有序分类任务上有根本性缺陷
- 序数回归 + BCE 是评分类任务的标配方案
- OOF 是唯一可信的评估方式
- 4 张 A100 40GB × 20h ≈ 够跑完一套完整消融
