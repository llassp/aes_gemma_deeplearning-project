# V8 完整方案技术详解

## Kaggle AES 2.0 — 基于序数回归与注意力池化的作文自动评分

---

## 一、任务背景

### 1.1 赛题

[Kaggle AES 2.0](https://www.kaggle.com/competitions/learning-agency-lab-automated-essay-scoring-2) 要求模型对 6-12 年级学生的英语作文进行自动评分，分数范围 **1~6 分**（1 分最差，6 分最优）。

### 1.2 评估指标

**Quadratic Weighted Kappa (QWK)**：衡量预测分数与真实分数之间的一致性，对偏离程度进行平方惩罚。QWK = 1.0 表示完全一致，0 表示随机一致。

### 1.3 核心挑战

1. **极端类别样本稀缺**：5 分和 6 分作文分别仅占 ~6% 和 ~1%
2. **"向均值收缩"问题**：回归模型天然倾向于预测中间分数（2-4 分）
3. **长文本处理**：部分作文超过 1500 tokens，需要有效的池化策略
4. **分数间有序关系**：1→2→3→4→5→6 是天然的有序分类任务

---

## 二、基座模型：Gemma 4 E4B + LoRA

### 2.1 为什么选择 Gemma 4？

- Google 最新的 E4B（~40 亿参数）多模态模型
- 在 A100 40GB ×4 上可以在 BF16 精度下完成 LoRA 微调
- 对英文文本有原生 tokenizer 优化

### 2.2 为什么使用 LoRA？

LoRA (Low-Rank Adaptation) 在冻结原始权重的前提下，为每个注意力层插入可训练的低秩矩阵，大幅降低训练参数量：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `r` | 16 | 低秩维度，越大微调能力越强 |
| `lora_alpha` | 32 | 缩放因子，控制 LoRA 输出幅度 |
| `lora_dropout` | 0.05 | 防止过拟合 |
| `target_modules` | q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj | 覆盖全部线性层 |

最终可训练参数量约 **53.7M**，仅占总参数（~7.95B）的 **0.68%**。

## 四、改进 1：Huber 回归 → 序数回归（Ordinal Regression）

### 4.1 问题：为什么回归不适合评分任务？

V6 使用 Huber Loss 做标量回归（输出 1 个浮点数，映射到 0.0~5.0）。这存在根本性问题：

- 把 6 个离散评分当作连续值，忽略了它们之间的**有序区间关系**
- MSE/Huber 对极端值施加巨大梯度，迫使模型"保守预测"——即向均值收缩
- 模型永远不敢预测 1 分或 6 分，因为一旦预测错误损失极大

### 4.2 方案：5 维序数目标

将评分 1~6 转换为 5 个二分类问题（5 个"切点"）：

```
score = 1 → [0, 0, 0, 0, 0]  (不大于任何切点)
score = 2 → [1, 0, 0, 0, 0]  (大于切点1)
score = 3 → [1, 1, 0, 0, 0]  (大于切点1,2)
score = 4 → [1, 1, 1, 0, 0]
score = 5 → [1, 1, 1, 1, 0]
score = 6 → [1, 1, 1, 1, 1]  (大于全部5个切点)
```

**关键直觉**：每个切点是一个独立的二分类——"这篇作文是否 > k 分？"。5 个切点共享同一套 backbone 特征，但各自的决策边界独立，不再受标量回归的"均值引力"约束。

### 4.3 损失函数：BCEWithLogitsLoss + pos_weight

```python
# 输出 logits: [B, 5] → sigmoid → 5个独立概率
# 目标 ordinal_labels: [B, 5] → 5个0/1标签

loss = BCEWithLogitsLoss(logits, ordinal_labels, pos_weight=pos_weight)
```

**pos_weight** 是关键：5 个切点各自的 pos_weight 从训练集计算：

```python
# 切点 k (分数 > k) 的正样本权重
pos_weight[k] = neg_count / pos_count

# 实际计算值（示例）：
# 切点1 (score>1): pos_weight ≈ 1.2   — 大部分样本都>1分
# 切点4 (score>4): pos_weight ≈ 4.0   — 只有~7%的样本>4分，需要加权
# 切点5 (score>5): pos_weight ≈ 9.0   — 只有~1%的样本>5分，需要大幅加权
```

**效果**：模型在高分切点（切点 4-5）上得到更大的梯度信号，不再被低频高分样本"淹没"。

### 4.4 推理时：logits → 连续分数

```python
def logits_to_continuous_score(logits):
    # sigmoid 得到每个切点的概率
    probs = sigmoid(logits)        # [B, 5]
    # 累加概率 + 基础分1 → 连续分数 [1.0, 6.0]
    return probs.sum(dim=1) + 1.0
```

例如：logits = `[2.0, 1.5, 0.3, -0.5, -2.0]`
→ probs = `[0.88, 0.82, 0.57, 0.38, 0.12]`
→ score = 0.88 + 0.82 + 0.57 + 0.38 + 0.12 + 1.0 = **3.77**

## 五、改进 2：Last-Token → Attention Pooling

### 5.1 问题：Last-Token Pooling 的局限

V6 取序列中最后一个非 padding token 的 hidden state 作为整篇作文的表示。这存在明显缺陷：

- 一篇 1500 tokens 的作文，只有最后一个 token 的信息被用到
- 如果关键信息在前半部分（如开头的论点、中间的论证），直接被丢弃
- 长文本中的错误线索（语法错误、拼写错误）散布在各处，需要全局感知

### 5.2 方案：可学习的注意力加权

```python
class AttentionPooling:
    """
    对全部 token 做加权平均，权重由 4 层 MLP 学习
    """
    def __init__(self, hidden_size):
        self.attention = Sequential(
            Linear(H→H),      # 特征变换
            LayerNorm(H),     # 稳定训练
            GELU(),           # 非线性
            Linear(H→1)        # 标量注意力分数
        )
        # 关键初始化：Linear(H→1) 的 weight 和 bias 全零
        # → 初始注意力 ≈ 均匀分布 → 等价于 Mean Pooling

    def forward(self, hidden_states, attention_mask):
        w = self.attention(hidden_states)   # [B, L, 1]
        w[attention_mask == 0] = -inf       # 屏蔽 <pad> token
        w = softmax(w, dim=1)               # 归一化
        return (w * hidden_states).sum(dim=1)  # 加权平均
```

### 5.3 为什么零初始化是关键？

- 如果不做零初始化，随机权重的注意力会极度偏向某个 token（通常是第一个或最后一个）
- 导致训练起步阶段 loss 极大甚至发散
- 零初始化 → 初始注意力均匀分布 → 等价于 Mean Pooling → 平稳起步 → 逐渐学习有意义的注意力模式

## 六、改进 3：5-Fold OOF + 阈值优化

### 6.1 为什么需要 OOF？

OOF (Out-of-Fold) 是衡量模型泛化能力的最诚实指标：

```
数据 → 随机分5份
Fold 0: [Val] [Train][Train][Train][Train] → 预测 Val
Fold 1: [Train][Val] [Train][Train][Train] → 预测 Val
...
每个样本都被"没见过它的模型"预测过一次
```

最终 17,307 个 OOF 预测构成对模型真实泛化水平的无偏估计。

### 6.2 阈值优化

序数回归输出的是连续分数（1.0~6.0），需转换为离散的 1~6 分。固定使用 `[1.5, 2.5, 3.5, 4.5, 5.5]` 会损失 QWK。

**两步优化**（在 OOF 预测上执行）：
1. **Nelder-Mead**: 从 `[1.5, 2.5, 3.5, 4.5, 5.5]` 出发，优化 5 个阈值使 QWK 最大化
2. **Brute-Force**: 在 5→6 边界（threshold[4]）做 0.01 步长的暴力搜索

```
优化前 (固定阈值): QWK ≈ 0.82
优化后: QWK = 0.8354
阈值: [1.63, 2.57, 3.59, 4.77, 5.40]
                          ↑
              5→6 边界降至 5.40
         （放宽高分判定标准，鼓励模型给5-6分）
```

## 七、改进 4：两阶段渐进训练

### 7.1 动机

新添加的 Attention Pooling 和 Ordinal Head 是随机初始化的，而 LoRA adapter 继承自预训练权重。如果同时训练，head 初期的较大梯度会"污染" LoRA 参数。

### 7.2 方案

```
Stage 1: Freeze LoRA → 仅训练 Head + Pooler (1 epoch)
         ├─ 学习率: 5e-4 (高)
         ├─ 可训练参数: 13.15M
         └─ 目标: 让 head 输出范围稳定

Stage 2: Unfreeze LoRA → 联合训练 (2 epochs)
         ├─ LoRA 学习率: 1e-4
         ├─ Head+Pooler 学习率: 1e-5 (0.1×)
         ├─ 可训练参数: 53.74M
         └─ 目标: 精细联合优化
```

## 十一、工程细节

### 11.1 Gemma4 PEFT 兼容性

Gemma4 使用自定义 `ClippableLinear` 层，不暴露标准的 `.weight` 属性，导致 PEFT 保存 adapter 时失败。解决方式：

```python
def patch_gemma4_clippable_layers(model):
    # 动态为 ClippableLinear 类添加 .weight property
    for module in model.named_modules():
        if "ClippableLinear" in module.__class__.__name__:
            cls = module.__class__
            @property
            def weight_property(self):
                # 从 named_parameters 中查找权重参数
                ...
            cls.weight = weight_property
```

### 11.2 DDP 多卡训练

- 使用 `torchrun --nproc_per_node=4` 启动
- 通过 `device_map={"": f"cuda:{local_rank}"}` 显式绑定 GPU
- `ddp_find_unused_parameters=true` 确保 pooler 参数正确同步

### 11.3 统一 Prompt 模板

训练和推理使用**完全相同**的模板，避免分布漂移：

```
<bos><start_of_turn>user
You are an expert essay grader. Read the student's essay and assign
a holistic score on a scale of 1 to 6, where 1 = very poor and
6 = excellent. Consider organization, development, sentence structure,
word choice, voice, and conventions.

Essay:
{raw essay text}<end_of_turn>
<start_of_turn>model
```
